# 🍜 SQL JOIN 超入門ガイド

> **「JOIN ってなに？」** を、日常の例え話を使って、ゼロからわかるように解説します。

---

## 🤔 そもそも JOIN ってなに？

**ひとことで言うと：「2つの表をくっつけること」** です。

### 日常の例え話

あなたはラーメン屋の店長です。手元に2つのノートがあります。

📒 **お客さんノート**（誰が来たか）

| お客さんID | 名前      |
|-----------|----------|
| 1         | 田中くん   |
| 2         | 佐藤さん   |
| 3         | 鈴木くん   |

📒 **注文ノート**（何を頼んだか）

| 注文ID | お客さんID | メニュー   |
|-------|-----------|----------|
| 101   | 1         | 味噌ラーメン |
| 102   | 2         | 醤油ラーメン |
| 103   | 1         | 餃子      |

この2つのノートを **「お客さんID」を手がかりにくっつける** と……

| 名前     | メニュー     |
|---------|------------|
| 田中くん  | 味噌ラーメン  |
| 佐藤さん  | 醤油ラーメン  |
| 田中くん  | 餃子        |

👆 **これが JOIN です！** 2つのバラバラの表を、共通の列（お客さんID）で合体させました。

---

## 📝 JOIN の基本の書き方

```sql
SELECT 欲しい列
FROM テーブルA
JOIN テーブルB ON テーブルA.共通の列 = テーブルB.共通の列;
```

さっきのラーメン屋の例だと：

```sql
SELECT customers.name, orders.menu
FROM customers
JOIN orders ON customers.customer_id = orders.customer_id;
```

**読み方：**
1. `FROM customers` → 「お客さんノートから」
2. `JOIN orders` → 「注文ノートをくっつけて」
3. `ON customers.customer_id = orders.customer_id` → 「お客さんIDが同じもの同士を繋げてね」
4. `SELECT customers.name, orders.menu` → 「名前とメニューだけ見せて」

## 🎯 JOIN は4種類ある

ここが一番大事なところです。  
**違いは「一致しないデータをどうするか」** だけです。

以下のサンプルデータで説明します。

📒 **お客さんテーブル**

| customer_id | name    |
|-------------|---------|
| 1           | 田中くん  |
| 2           | 佐藤さん  |
| 3           | 鈴木くん  | ← 注文してない！

📒 **注文テーブル**

| order_id | customer_id | menu       |
|----------|-------------|------------|
| 101      | 1           | 味噌ラーメン |
| 102      | 2           | 醤油ラーメン |
| 103      | 99          | チャーハン   | ← お客さんテーブルにいない！

---

### ① INNER JOIN（一致するものだけ）

**「両方にあるデータだけ見せて」**

```
お客さん       注文
┌─────┐     ┌─────┐
│  1  │──── │  1  │  ✅ 両方にある → 表示！
│  2  │──── │  2  │  ✅ 両方にある → 表示！
│  3  │     │     │  ❌ 注文がない → 消える
│     │     │ 99  │  ❌ お客さんがいない → 消える
└─────┘     └─────┘
```

```sql
SELECT *
FROM customers
INNER JOIN orders ON customers.customer_id = orders.customer_id;
```

**結果：**

| name    | menu       |
|---------|------------|
| 田中くん  | 味噌ラーメン |
| 佐藤さん  | 醤油ラーメン |

> 💡 **いつ使う？**  
> 「注文したお客さんの一覧が欲しい」みたいに、**確実に両方にあるデータだけ**欲しいとき。

---

### ② LEFT JOIN（左のテーブルを全部残す）

**「左の表は全部見せて。右にデータがなかったら NULL（空欄）にして」**

```
お客さん(左)    注文(右)
┌─────┐       ┌─────┐
│  1  │────── │  1  │  ✅ くっつく
│  2  │────── │  2  │  ✅ くっつく
│  3  │── ? ──│     │  ⚠️ 右にない → NULL で表示
│     │       │ 99  │  ❌ 左にない → 消える
└─────┘       └─────┘
```

```sql
SELECT *
FROM customers
LEFT JOIN orders ON customers.customer_id = orders.customer_id;
```

**結果：**

| name    | menu        |
|---------|-------------|
| 田中くん  | 味噌ラーメン  |
| 佐藤さん  | 醤油ラーメン  |
| 鈴木くん  | **NULL**    | ← 注文なし。でも名前は出る！

> 💡 **いつ使う？**  
> 「全お客さんの一覧を出して、注文してない人も含めて見たい」とき。  
> **一番よく使う JOIN** です。

#### 🔥 応用：注文してない人だけ見つける

```sql
SELECT customers.name
FROM customers
LEFT JOIN orders ON customers.customer_id = orders.customer_id
WHERE orders.order_id IS NULL;
```

結果 → `鈴木くん` だけ出る！

---

### ③ RIGHT JOIN（右のテーブルを全部残す）

**LEFT JOIN の逆。「右の表は全部見せて」**

```
お客さん(左)    注文(右)
┌─────┐       ┌─────┐
│  1  │────── │  1  │  ✅ くっつく
│  2  │────── │  2  │  ✅ くっつく
│  3  │       │     │  ❌ 右にない → 消える
│     │── ? ──│ 99  │  ⚠️ 左にない → NULL で表示
└─────┘       └─────┘
```

```sql
SELECT *
FROM customers
RIGHT JOIN orders ON customers.customer_id = orders.customer_id;
```

**結果：**

| name      | menu       |
|-----------|------------|
| 田中くん    | 味噌ラーメン |
| 佐藤さん    | 醤油ラーメン |
| **NULL**  | チャーハン   | ← お客さん不明。でも注文は出る！

> 💡 **いつ使う？**  
> 正直あまり使いません。LEFT JOIN でテーブルの順番を入れ替えれば同じことができるからです。

---

### ④ FULL OUTER JOIN（両方全部残す）

**「とにかく全部見せて。一致しなくても NULL で埋めて」**

```
お客さん(左)    注文(右)
┌─────┐       ┌─────┐
│  1  │────── │  1  │  ✅ くっつく
│  2  │────── │  2  │  ✅ くっつく
│  3  │── ? ──│     │  ⚠️ NULL で表示
│     │── ? ──│ 99  │  ⚠️ NULL で表示
└─────┘       └─────┘
```

```sql
SELECT *
FROM customers
FULL OUTER JOIN orders ON customers.customer_id = orders.customer_id;
```

**結果：**

| name      | menu        |
|-----------|-------------|
| 田中くん    | 味噌ラーメン  |
| 佐藤さん    | 醤油ラーメン  |
| 鈴木くん    | **NULL**    |
| **NULL**  | チャーハン    |

> ⚠️ **注意：MySQL では使えません！**  
> 代わりに LEFT JOIN と RIGHT JOIN を UNION で組み合わせます。

## 🗺️ 4つの JOIN を一枚の図で理解する

```
  INNER JOIN          LEFT JOIN           RIGHT JOIN        FULL OUTER JOIN
 （共通だけ）       （左を全部）         （右を全部）        （全部）

  ┌───┬───┐          ┌───┬───┐          ┌───┬───┐         ┌───┬───┐
  │   │███│          │███│███│          │   │███│         │███│███│
  │   │███│          │███│███│          │   │███│         │███│███│
  │   │███│          │███│███│          │   │███│         │███│███│
  └───┴───┘          └───┴───┘          └───┴───┘         └───┴───┘
     ↑                  ↑                  ↑                 ↑
  一致する部分       左テーブル全部      右テーブル全部       全部
  だけ返す         ＋一致する部分      ＋一致する部分      全部返す

  █ = 結果に含まれる部分
```

## 🏃 SELF JOIN（自己結合）おまけ

同じテーブルを自分自身とくっつけます。ちょっと上級者向け。

**例：** 社員テーブルから「誰の上司が誰か」を表示する

```sql
SELECT
    部下.name AS 部下の名前,
    上司.name AS 上司の名前
FROM employees 部下
JOIN employees 上司 ON 部下.boss_id = 上司.employee_id;
```

> 💡 同じテーブルに別名（エイリアス）をつけて区別します。

## ⚡ 初心者がよくやるミス

### ❌ ミス1：ON を書き忘れる

```sql
-- これだと全組み合わせが出てきてヤバいことになる
SELECT * FROM customers, orders;
```

3人 × 3件 = 9行 出てくる 😱

### ❌ ミス2：結合するカラムを間違える

```sql
-- customer_id 同士で結合すべきなのに…
SELECT * FROM customers
JOIN orders ON customers.customer_id = orders.order_id;  -- ← 違う列！
```

### ✅ コツ：テーブルに短い別名をつける

```sql
-- c = customers, o = orders
SELECT c.name, o.menu
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id;
```

こう書くと短くて読みやすい！

---

## 📋 まとめ：どれを使えばいい？

| やりたいこと                       | 使う JOIN         |
|----------------------------------|------------------|
| 両方にあるデータだけ欲しい             | `INNER JOIN`     |
| 左のテーブルは全部残したい            | `LEFT JOIN`      |
| 右のテーブルは全部残したい            | `RIGHT JOIN`     |
| とにかく全部見たい                  | `FULL OUTER JOIN`|

**迷ったら `LEFT JOIN` を使っておけばだいたい OK です。**
